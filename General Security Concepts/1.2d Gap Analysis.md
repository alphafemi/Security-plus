

# Gap Analysis

## Overview

A **gap analysis** is a study that compares:

- **Where we are** (current security posture)
- **Where we want to be** (target security baseline)

In IT security, gap analyses help organizations understand what security measures are needed and how to implement them.

> ⚠️ **Note**: While simple to explain, gap analysis is a complex process that can take weeks, months, or even years to complete.

---

## Why Perform a Gap Analysis?

| Benefit | Description |
|---------|-------------|
| Identify weaknesses | Understand existing system vulnerabilities |
| Plan improvements | Create a roadmap from current to target state |
| Prioritize actions | Focus resources on the most critical gaps |
| Meet compliance | Align with regulatory or industry baselines |

---

## Step 1: Establish a Baseline

A **baseline** provides a target to work toward. Common baselines include:

| Baseline | Description |
|----------|-------------|
| **NIST SP 800-171 Rev 2** | *Protecting Controlled Unclassified Information in Nonfederal Systems and Organizations* |
| **ISO/IEC 27001** | Information security management systems (International standards) |
| **Custom baseline** | Created specifically for your organization's needs |

### Baseline Evaluation Areas

| Area | Questions to Ask |
|------|------------------|
| **People** | What formal experience do they have? What training received? Do they know security policies? |
| **Processes** | Are existing IT systems aligned with formal security policies? |

---

## Step 2: Perform the Analysis

The analysis involves:

1. **Comparing** existing systems against the chosen baseline
2. **Identifying** weaknesses in current systems
3. **Researching** effective processes to compensate for weaknesses
4. **Breaking down** broad security categories into smaller, manageable segments

### Example: Breaking Down Access Control (NIST 800-171)

```
Access Control (Broad Category)
    │
    ├── Limit system access to unauthorized users
    ├── Limit access to processes acting on behalf of users
    ├── Limit access to devices
    │
    └── Account Management (Detailed controls)
           ├── User registration and deregistration
           ├── User access provisioning
           ├── Management of privileged access rights
           └── Review of user access rights
```

By breaking down broad areas, you can assess each individual security task.

---

## Step 3: Create the Gap Analysis Report

The final document summarizes:

- Current state vs. baseline objectives
- The path from current to target (including time, cost, equipment, change control)

### Color Legend

| Color | Meaning |
|-------|---------|
| 🟢 Green | Close to meeting baseline |
| 🟡 Yellow | Partially meeting baseline (midpoint) |
| 🔴 Red | Needs significant work |

> **Prioritization Tip**: Start with 🔴 red items for biggest security impact, then 🟡 yellow, then 🟢 green.

## The Gap Analysis Process Flow

```
┌─────────────────┐
│  Choose Baseline│  (NIST, ISO, or custom)
└────────┬────────┘
         ▼
┌─────────────────┐
│  Assess People  │  (Experience, training, policy knowledge)
│  & Processes    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Identify       │  (Compare existing systems to baseline)
│  Weaknesses     │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Break Down     │  (Broad categories → individual controls)
│  Security Areas │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Create Report  │  (Summary + detailed roadmap)
└─────────────────┘
```

---

## Key Takeaways

| Concept | Description |
|---------|-------------|
| **Gap Analysis** | Comparing current state to desired baseline |
| **Baseline** | Target standard (NIST, ISO, or custom) |
| **People & Processes** | Both must be evaluated |
| **Breaking Down** | Broad categories into individual controls |
| **Report** | Includes summary tables, colors, and implementation roadmap |
| **Path Forward** | Time, money, equipment, and change control required |
