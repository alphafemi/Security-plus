# Obfuscation

## Overview

**Obfuscation** is the process of taking information that is easy to understand and making it deliberately difficult to interpret — while keeping it in plain sight. Unlike encryption, obfuscation does not require a mathematical key to reverse; if you know the method used to obscure the data, you can recover the original.

> **Note:** Obfuscation alone is generally not considered true security. It relies on the obscurity of the method rather than the strength of a key or algorithm. It is most effective when used alongside other security controls.

---

## Steganography

**Steganography** (from the Greek for "concealed writing") hides data inside another file or medium so that the existence of the hidden data is not apparent to a casual observer.

The file or object used to carry the hidden data is called the **covertext**.

### Types of Steganography

| Medium | Description |
|--------|-------------|
| **Image** | Data is embedded within the pixel values of an image file. The image looks identical to the naked eye. |
| **Network traffic** | Messages are embedded within TCP packet fields and reconstructed on the receiving end. |
| **Audio** | Data is hidden within an audio file or track. |
| **Video** | Data is concealed within a video file, which can carry a large volume of hidden information. |
| **Printed documents** | Tiny dots printed on paper (nearly invisible to the eye) encode identifying information. |

### Machine Identification Codes

Many laser printers embed **machine identification codes** as a pattern of tiny yellow dots across every printed page. These dots are nearly invisible on white paper but become visible when the image is inverted or viewed under blue light.

If the dot pattern is known, it can be decoded to identify:

- The specific printer used to produce the document
- Potentially the date and time of printing

This is a real-world, built-in form of steganography used by printer manufacturers.

## Tokenization

**Tokenization** replaces sensitive data with a non-sensitive substitute called a **token**. The token has no mathematical relationship to the original value, making it meaningless if intercepted.

A lookup table on a secure server maps each token back to the original data. Only authorized systems can perform that reverse lookup.

### Properties of Tokens

- Cannot be reverse-engineered to reveal the original data
- Often single-use (one-time tokens)
- Can be transmitted across a network **without encryption**, since they carry no exploitable value

### Credit Card Tokenization: How It Works

Tokenization is the process behind mobile and smartwatch payments (e.g., Apple Pay, Google Pay):

1. **Registration** — The user registers a credit card on their device. The device contacts a **Token Service Server (TSS)**, which generates a set of tokens and stores them on the device.
2. **Checkout** — At the point of sale, the device transmits a token (not the real card number) via near-field communication (NFC).
3. **Validation** — The merchant forwards the token to the TSS, which looks up the real card number and validates the transaction.
4. **Token disposal** — Once used, the token is discarded and cannot be reused. The device prepares the next token for future transactions.

```
User's Device          Merchant          Token Service Server
     |                    |                      |
     |-- Token (NFC) ---> |                      |
     |                    |-- Token -----------> |
     |                    |                      | (reverse lookup)
     |                    | <-- Approval ------- |
     |                    |                      |
     | (token discarded)  |                      |
```

> If an attacker captures a token in transit, it is already invalid by the time they attempt to use it.

## Data Masking

**Data masking** hides portions of sensitive data while displaying just enough to be useful in context. The full value is known and stored by the authoritative system but is not exposed unnecessarily.

### Common Examples

- **Receipts** — Credit card numbers are printed as `**** **** **** 2512`, showing only the last four digits
- **Customer service** — Representatives may only see the last four digits of an account number when verifying identity
- **Internal systems** — Employees working with customer data see masked values unless they have explicit access rights

### Masking Techniques

- Replacing characters with asterisks or Xs
- Rearranging digits in a reversible pattern
- Substituting digits with alternate values that can be reversed by an authorized process

---

## Comparison

| Technique | How It Hides Data | Reversible Without a Key? | Common Use Case |
|-----------|------------------|--------------------------|-----------------|
| **Steganography** | Embeds data inside another file or medium | Yes, if method is known | Covert communication, watermarking |
| **Tokenization** | Replaces data with an unrelated substitute | Only via secure lookup server | Payment processing, PII protection |
| **Data masking** | Partially conceals a value | Yes, by authorized systems | Receipts, customer service, internal tools |

---

## Summary

| Concept | Key Takeaway |
|---------|-------------|
| **Obfuscation** | Hides data in plain sight; security depends on method remaining unknown |
| **Steganography** | Conceals data within images, audio, video, network traffic, or printed documents |
| **Machine identification codes** | Yellow dot patterns embedded by printers to identify the source device |
| **Tokenization** | Replaces sensitive data with a one-time, non-reversible substitute; safe to transmit without encryption |
| **Data masking** | Partially obscures sensitive values while preserving usability in limited contexts |
