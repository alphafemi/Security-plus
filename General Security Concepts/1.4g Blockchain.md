# Blockchain

## Overview

A **blockchain** is a distributed ledger — a record of transactions that is shared openly and maintained simultaneously by everyone participating in the network. No single person or organization owns or controls it; instead, every participant holds a full copy and helps keep it up to date.

---

## Key Characteristics

| Property | Description |
|----------|-------------|
| **Distributed** | Every participant maintains their own copy of the ledger |
| **Transparent** | The ledger is available for anyone on the network to view |
| **Tamper-evident** | Any modification to a recorded block invalidates its hash, alerting the network |
| **Decentralized** | No single authority controls the record — consensus is maintained across all participants |

---

## Common Use Cases

Blockchain is useful for any application that requires a trustworthy, auditable record of transactions:

- **Payment processing** — cryptocurrency transfers (e.g., Bitcoin)
- **Digital identification** — verifying identity without a central authority
- **Supply chain monitoring** — tracking goods from origin to destination
- **Digital voting** — recording votes in a tamper-resistant ledger
- **Data backup tracking** — logging and verifying the transfer of backup data
- **Title and asset transfer** — recording the transfer of property or ownership documents

---

## How a Blockchain Transaction Works

### 1. A Transaction Is Initiated

A transaction is created — for example, transferring Bitcoin from one party to another, or recording the transfer of a house title.

### 2. The Transaction Is Broadcast to the Network

Rather than sending the transaction to a single central server, it is distributed to **every participant** (node) on the blockchain network. Each node receives and stores a copy.

### 3. Transactions Are Grouped Into a Block

Incoming transactions are collected and bundled together into a **block**. A block is essentially a batch of pending transactions waiting to be permanently recorded.

### 4. A Hash Is Added to the Block

Once the block is complete, a **cryptographic hash** is calculated from all of the transactions it contains. This hash acts as a seal of integrity for the block.

```
[Transaction 1]
[Transaction 2]  ──► [Hash Algorithm] ──► Block Hash
[Transaction 3]
```

### 5. The Block Is Distributed and Added to the Chain

The completed block — including its hash — is sent to all participants. Each node adds it to their local copy of the ledger, extending the chain.

---

## Tamper Detection via Hashing

The hash attached to each block is what makes the blockchain tamper-evident. If anyone modifies a transaction inside an already-recorded block, the hash of that block will no longer match its contents.

```
Original Block:   [Tx1][Tx2][Tx3] ──► Hash: a1b2c3...   ✅ Valid
Tampered Block:   [Tx1][Tx2*][Tx3] ──► Hash: 9f4e71...  ❌ Hash mismatch
```

When other nodes on the network receive a block with an invalid hash, they **reject it** and flag it as tampered. Because every participant holds a copy of the ledger, a bad actor would need to simultaneously alter the records on the majority of all participating nodes — a practically infeasible task on a large network.

## Summary

| Concept | Key Takeaway |
|---------|-------------|
| **Distributed ledger** | All participants hold and maintain a copy of the full transaction record |
| **Block** | A batch of transactions bundled together for processing |
| **Hash** | A cryptographic fingerprint added to each block to ensure integrity |
| **Tamper detection** | Modifying any recorded transaction invalidates the block's hash, alerting the network |
| **Decentralization** | No single point of failure or control — consensus is maintained across all nodes |
