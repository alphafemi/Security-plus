# Threat Vectors

## Overview

A **threat vector** (also called an **attack vector**) is the method an attacker uses to gain access to a system. Attackers continuously discover and develop new vectors, targeting both well-known weaknesses and previously unknown opportunities.

---

## Messaging-Based Vectors

Messaging systems are among the most frequently exploited entry points because they provide direct access to users and can bypass many technical controls through social engineering.

| Channel | Attack Method |
|---------|--------------|
| **Email** | Malicious links, phishing pages, malware attachments, fake invoices, cryptocurrency scams |
| **SMS / Text** | Smishing — links disguised as delivery notifications, bank alerts, or prize claims |
| **Instant / Direct Message** | Direct social engineering, malicious links embedded in conversation |

### How Phishing Works via Messaging

1. Attacker sends a message with an enticing or urgent hook
2. Victim clicks a link and is taken to a convincing but fake page (e.g., a bank login)
3. Credentials are captured, or malware is downloaded in the background

> **Real-world example:** SMS messages impersonating the US Postal Service — "Your package is on hold due to an incorrect address. Click here to reschedule." — are a common smishing template. The link leads to a credential-harvesting or malware-delivery page.

## Image-Based Vectors

Images can serve as attack vectors when they contain embedded code rather than just visual data.

### SVG (Scalable Vector Graphics)

SVG files are XML documents that *describe* an image rather than storing pixel data. This means they can contain:

- Embedded HTML
- Executable JavaScript

When a browser renders an SVG, any embedded scripts can execute — making SVGs a potential **cross-site scripting (XSS)** delivery mechanism. Most modern browsers include protections against this, but unpatched or misconfigured browsers remain vulnerable.

## File-Based Vectors

Many common file formats can carry malicious payloads:

| File Type | Risk |
|-----------|------|
| **Executables (.exe, .bin)** | Directly run malicious code in memory |
| **PDFs** | Can contain embedded scripts and objects |
| **Compressed archives (.zip, .rar)** | Obfuscate malicious files within many legitimate ones |
| **Office documents (.docx, .xlsx)** | Can include macros that exfiltrate data or execute commands |
| **Browser extensions / add-ins** | Malicious extensions run with elevated browser privileges |

### Office Macros

Microsoft Office allows documents to include **macros** — small programs that automate tasks. Attackers can write macros that silently gather system information and transmit it to an external server. Macro-based attacks remain common because users are accustomed to enabling macros for legitimate purposes.

## Voice & Phone-Based Vectors

| Method | Description |
|--------|-------------|
| **Vishing** | Voice phishing — calling victims to extract credit card numbers or personal details |
| **SPIT** | Spam over IP Telephony — automated mass spam calls using VoIP systems |
| **War dialing** | Systematically calling phone numbers to find modems or systems connected to phone lines |
| **Denial-of-service via phone** | Flooding phone or messaging systems to disrupt communications |

## Physical Vectors

### USB Drives

A USB drive can bypass millions of dollars of network security controls entirely. Common techniques include:

- **Parking lot drop** — leaving infected USB drives where employees will find and plug them in
- **HID spoofing** — a USB device that identifies itself as a keyboard, automatically typing malicious commands when plugged in
- **Data exfiltration** — copying large amounts of data onto a USB drive and walking out the door

USB attacks are especially effective against **air-gapped networks** — networks with no external internet connection — where physical media is the only way data moves in or out.

## Software & Application Vectors

### Vulnerable or Outdated Software

Every unpatched vulnerability is an open door. Attackers actively search for known vulnerabilities in software that has not been updated. Key risks include:

- **Locally installed applications** with unpatched security flaws
- **Web-based / agentless applications** — if the server is compromised, every connecting client is at risk
- **Unsupported operating systems** — once a vendor stops releasing patches, any newly discovered vulnerability will never be fixed

> Organizations should maintain a current inventory of all systems on the network. Unsupported systems running on forgotten hardware — such as an old machine under someone's desk — represent significant unmanaged risk.

## Network Infrastructure Vectors

The internal network itself can be exploited as a highway for lateral movement once an attacker gains initial access.

| Vector | Risk and Mitigation |
|--------|-------------------|
| **Outdated wireless protocols** | WEP, WPA, and WPA2 have known weaknesses — upgrade to **WPA3** |
| **Rogue access points** | Unauthorized wireless APs can provide attackers easy network entry — scan periodically |
| **Bluetooth** | Can be used for reconnaissance or exploited if implementation has security flaws |
| **Open ports** | Every running service opens a potential entry point — use port-based or application-aware firewalls to limit exposure |
| **Misconfigured services** | Complex applications are easy to misconfigure — a single error can grant unauthorized access |

### 802.1X Authentication

Enabling **802.1X** on both wired and wireless networks requires devices to authenticate before being granted network access, preventing unauthorized devices from joining the network at all.

---

## Default Credentials

Many routers, switches, access points, and other devices ship with well-known default usernames and passwords (e.g., `admin` / `admin`). If these are not changed, anyone who knows the defaults has full administrative access to the device.

Default credentials for thousands of devices are publicly documented on sites such as `routerpasswords.com`.

**Best practice:** Change default credentials immediately upon first login. Many modern devices now enforce this during initial setup.

## Supply Chain Vectors

Attackers can embed threats into products or relationships before they ever reach your organization.

| Supply Chain Attack | Description |
|--------------------|-------------|
| **Compromised hardware** | Malicious code added during or after manufacturing |
| **Counterfeit hardware** | Fake devices that may contain backdoors or fail to apply security updates |
| **Managed Service Provider (MSP)** | Attackers compromise an MSP to gain access to all of its clients simultaneously |

### Real-World Examples

**Target (2013):** Attackers gained access to Target's network through credentials belonging to an HVAC contractor. From the HVAC management system, they pivoted into the retail network and installed malware on point-of-sale systems across all Target stores, stealing millions of credit card numbers.

**Fake Cisco Switches (2020):** Counterfeit Cisco Catalyst switches were identified after they failed to update their firmware correctly. Such devices could easily be used to intercept traffic or provide persistent backdoor access.

---

## Summary

| Threat Vector | Key Risk |
|--------------|----------|
| **Messaging (email, SMS, IM)** | Phishing, malware delivery, social engineering |
| **Images (SVG)** | Embedded scripts that execute in the browser |
| **Files (PDF, Office, archives)** | Malicious payloads hidden in common file formats |
| **Voice / phone** | Vishing, spam calls, war dialing |
| **USB / physical media** | Bypasses network controls entirely; enables air-gap attacks |
| **Unpatched software** | Known vulnerabilities exploited before or after patching |
| **Unsupported systems** | Permanently unpatched; persistent risk |
| **Network infrastructure** | Lateral movement, rogue APs, open ports, misconfigurations |
| **Default credentials** | Publicly known; grants immediate administrative access |
| **Supply chain** | Compromised hardware, software, or third-party vendor access |
